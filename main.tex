\begin{abstract}
\noindent
We formulate a static multi-trip routing and inter-trip charging problem for an Electrified-Autonomous Flexible Transit (E-AFT) fleet with multi-station choice and solve it as a single mixed-integer program. The bilinear charging relation \(E_{ksr}=p_{ksr}\theta_{ksr}\) is replaced by a station-specific piecewise-linear map from charging time to charged energy. On three synthetic scenarios, the linear and nonlinear charging assumptions diverge only in the partial-recharge case, where the nonlinear curve lets the bus serve one extra request and adds \$76.99 in profit. In the baseline and deep-recharge scenarios (the latter at \(R=5\)) the two assumptions yield the same served set and the same objective, and differ only in the charging time needed to acquire the same energy. The partial-recharge gap sits in the narrow regime where a moderate recharge becomes profitable only when the taper is exploited.
\end{abstract}

\section{Problem Description}

Electrified-Autonomous Flexible Transit (E-AFT) is an emerging and promising transportation mode for passenger travel. It allows passengers to submit travel requests online before traveling. They can detail their travel origins, destinations, and acceptable pickup time. Based on passenger requests, E-AFT routes are generated by selecting the appropriate stops to pick up and drop off passengers. Compared to traditional electric buses, E-AFT has a limited battery capacity that constrains traveling range. The smaller battery capacities make them more sensitive to routing planning to serve passenger requests, which requires further research on the optimization of transportation routing planning. In addition, the majority of current research assumes charging only happens at fixed depots \cite{gao2025integrated}. Based on these research gaps, Jiang et al. (2026) \cite{jiang2026two} developed a two-stage optimization approach for dynamic routing and charging scheduling. They assume that charging is restricted to happen between trips where no passengers are on board. They also assume a linear charging speed for vehicle charging. However, the charging speed can be nonlinear in reality \cite{yi2025charging, montoya2017electric}. Building on \cite{jiang2026two}, this project explores a static version of the trip routing and charging scheduling problem with a nonlinear charging profile and an explicit choice among multiple charging stations. The setting is illustrated in Figure~\ref{fig:concept}. Each bus moves from one stop to another to pick up and drop off passengers, which forms a trip. After all passengers drop off, the bus moves to a charging station to recharge. It then moves to the first bus stop of the next trip. The objective is to decide which passenger requests will be served by each bus, along with the route each bus follows within energy-consumption constraints. The nonlinear aspect of the project arises from the charging process, where the charging rate varies with the battery state instead of remaining constant. As a result, routing, timing, and charging decisions are tightly coupled.

\section{Mathematical Formulation}

Mathematical notation is summarized in Table~\ref{tab:notation}. Compared with the current study \cite{jiang2026two}, this project focuses on a static version in which routing and charging decisions are optimized once for the entire planning horizon. Instead of solving the problem in two heuristic stages, we formulate it in one optimization problem. In addition, this project examines the situation where the charging speed is nonlinear. For simplicity, several components of the original setting are omitted.

The objective function is the operational profit. It includes the revenue from served passenger requests, the fixed vehicle deployment cost, the penalty cost for unserved requests, the routing cost, and the charging cost.

Constraints (2) and (3) describe whether each request is served by a vehicle or not served. Constraints (4) and (5) ensure that the trip is formed, as at least one request is served during the trip. A subsequent trip can only be formed after the current trip is formed. Constraints (6)--(9) ensure that the pick-up and drop-off nodes are visited exactly once if the request is assigned to the trip. Constraint (10) ensures the flow conservation inside each trip. Constraints (11)--(14) define the first and the last node visited in trips by node connections. When there is no incoming arc for a node that is in the trip, it is the first node visited in the trip. If there is no outgoing arc for a node that is in the trip, it is the last node visited in the trip. Constraints (15)--(18) describe time constraints. Constraints (15) and (16) ensure that the arrival time at the pick-up node is within the acceptable time range. Constraint (17) ensures the drop-off time is later than the pick-up time. Constraint (18) describes within-trip time propagation. Constraints (19)--(21) represent the passenger-load changes as the bus picks up and drops off passengers along the way. Constraints (22)--(25) enforce the trip start and end time to equal the arrival time of the first and the last node of the trip. Constraint (26) ensures the arrival time of the starting node of the next trip is later than the arrival time of the last node of the current trip, plus the time traveling between nodes and a charging station, and the charging duration. Constraints (27) and (28) represent the change in battery level. Constraints (29)--(33) represent trip-level battery boundary constraints. Constraint (34) ensures that charging happens only between trips. Constraint (35) presents the relationship between charging power and charging time. Constraints (36) and (37) define the battery-level transitions between trips. Constraints (38)--(40) apply battery boundaries to the battery level along the trip. Constraints (41)--(51) pertain to the binary or non-negative variables. Among these constraints, (35) is a bilinear form that is naturally nonlinear, and the charging power \(p_{ksr}\) can itself be nonlinear.

\section{Implementation and Simplifications}

The model is implemented in Python with Gurobi.

\subsection{Why a Piecewise-Linear Charging Map Instead of the Bilinear Form}

Constraint (35), \(E_{ksr}=p_{ksr}\theta_{ksr}\), is bilinear in two continuous decision variables and, in the physically meaningful case, the charging power \(p_{ksr}\) is itself a function of the current battery state \(L^{\mathrm{start}}_{kr}\). Keeping this relation in its bilinear form inside the integrated model forces the solver into a mixed-integer quadratically constrained program (MIQCP) whose spatial branching interacts with the already large binary route-and-assignment space; in a size-limited license regime such as ours, this combination does not scale past two to three requests.

A more conservative relaxation -- McCormick envelopes -- would keep the problem MILP but yield only a convex outer approximation whose gap on tapering charging curves is driven by the largest product term, so solution quality is sensitive to the tightness of the battery-capacity bounds and the charging-time bounds used to build the envelope.

The formulation we adopt takes a different route. Physically, the quantity \(p_{ksr}\) is not an independent control; for a given battery and charger it is pinned to the current state of charge by the battery management system, and published fast-charge profiles for lithium-ion EV batteries follow a CC-CV pattern whose energy-versus-time curve is well approximated by three to four affine segments \cite{yi2025charging, montoya2017electric}. Our model therefore keeps the \emph{physically observable} quantity -- the total energy \(E_{ksr}\) acquired as a function of time \(\theta_{ksr}\) -- and drops \(p_{ksr}\) entirely as a redundant intermediate variable. The resulting relation is encoded as \(\mathrm{PWL}\bigl(\theta_{ksr}; \theta^{(1)}_s,\ldots,\theta^{(P)}_s; E^{(1)}_s,\ldots,E^{(P)}_s\bigr)\) via Gurobi's \texttt{addGenConstrPWL}. On a segmented CC-CV-style curve this representation is exact at the breakpoints and interpolates linearly between them. Linear charging corresponds to evenly spaced breakpoints with constant slope; nonlinear (tapering) charging uses concave, decreasing-slope breakpoints. Station differences are expressed by scaling the time axis of the breakpoints, so a ``hub'' station delivers the same energy in less time than a ``depot.'' Figure~\ref{fig:charging-profiles} shows both base curves and the three station-specific variants.

\subsection{Tractability Adjustments}

Three further changes keep the integrated formulation tractable under a size-limited Gurobi license. The piecewise-linear charging relation is wrapped in an indicator on \(z_{ksr}\) so that the breakpoint model is inert when a station is not chosen; the redundant visit indicator \(v_{kir}\) is substituted by the corresponding assignment variable; and the initial dispatch station is promoted to a decision variable, so the first trip of each bus selects its origin together with the rest of the plan. Solver-side adjustments (per-constraint tightened big-\(M\), hierarchical multi-objective with profit primary and completion time secondary, and lex-order symmetry breaking across identical buses) are documented in the code repository.

\section{Computational Study}

We evaluate the model on synthetic instances sized to fit within the current size-limited Gurobi license (roughly a few hundred variables and constraints). The default configuration uses one bus, two trip slots, two charging stations, and four passenger requests; the deep-recharge scenario uses five requests. Travel times and energy consumption are generated from Euclidean distances on a small synthetic grid.

We evaluate three representative scenarios:
\begin{itemize}
    \item \textbf{Baseline} -- a loose 4-request instance in which both charging models can serve all requests. This is the sanity check.
    \item \textbf{Partial recharge} -- a tighter 4-request instance with shifted time windows, designed so that a \emph{moderate} recharge is sufficient. This scenario is meant to probe whether the tapering curve helps when the bus mainly charges in the fast early portion of the curve.
    \item \textbf{Deep recharge} -- a tighter 5-request instance with further-shifted windows, so that fitting the fifth request into the second trip requires a deeper recharge between trips. Originally drafted at 6 requests, this scenario is scaled down here to fit within the size-limited license after the initial-station decision variables were added.
\end{itemize}

In the \textbf{baseline} case, both assumptions serve all four requests at objective \(202.98\). In the \textbf{partial-recharge} case, the two optima diverge: the linear assumption drops \(r_4\) and converges at \(125.99\) with a short \(2.29\)-min recharge (\(3.35\) kWh); the nonlinear assumption charges harder (\(4.52\) min, \(10.17\) kWh) and serves all four requests at \(202.98\). Neither assumption finds the fifth request profitable at this instance size; a sharper contrast would require returning to the 6-request instance, which exceeds our current license budget once the initial-station decision layer is active.

\subsection{Problem Size and Runtime}

Table~\ref{tab:solver-stats} reports the size of the integrated MIP as Gurobi receives it, i.e.\ before presolve, and the Gurobi wall-clock runtime for each scenario and charging mode. Model size is driven almost entirely by the arc variables \(x_{kijr}\) and the link variables \(\ell^{\text{last}}\), \(\ell^{\text{start}}\), which scale as \(|N|^{2}|R|\) with \(|N|=2|Q|\) service nodes. Trip slots \(|R|\) are fixed at two throughout; the scale difference across scenarios comes from varying \(|Q|\). The deep-recharge instance (\(|Q|=5\)) therefore carries roughly 50\% more constraints than the \(|Q|=4\) instances, and its runtime is about five times larger, still well under a second under Gurobi's default settings. The linear and nonlinear modes produce the same model size because the piecewise-linear charging relation has the same number of breakpoints on each curve; the runtime differences between modes are within noise.

\subsection{Structural Changes in the Optimal Plans}

The charging assumption changes more than the objective value. It also changes the discrete operational plan.

In the \textbf{partial-recharge} scenario:
\begin{itemize}
    \item the linear model serves \(r_1,r_2\) in Trip 1 and \(r_3\) alone in Trip 2, dropping \(r_4\);
    \item the nonlinear model serves \(r_1,r_2\) in Trip 1 and \(r_3,r_4\) in Trip 2 (all four served).
\end{itemize}

In the \textbf{deep-recharge} scenario, both models converge on the same grouping \(\{r_1,r_2\}\) in Trip 1 and \(\{r_3,r_4\}\) in Trip 2, dropping \(r_5\). When the tapering curve helps, it helps by enabling one additional request rather than by rearranging the trip grouping.

\subsection{Station Selection}

Across all runs the solver dispatches from \texttt{depot} and recharges at \texttt{hub}. The hub combines favorable repositioning geometry with the fastest time-scaling factor, so both the mid-plan station and the initial dispatch station end up actively selected rather than left at their default.

\subsection{Cross-Evaluation: Is the Linear-Optimal Plan Still Good Under Nonlinear Charging?}

We also test whether a plan optimized under the simplified linear charging assumption remains valid once the charging process is modeled more realistically. The cross-evaluation has three stages: (i) solve under linear charging; (ii) freeze the resulting discrete plan (routing, trip assignment, station choice) and re-solve the continuous timing and charging variables under nonlinear charging; (iii) re-optimize everything under nonlinear charging and compare.

The outcome separates cleanly into two effects. Holding the linear-optimal \emph{discrete plan} fixed, the nonlinear curve buys exactly the same energy in less time (\(1.32\) vs.\ \(2.29\) units in partial recharge; \(4.52\) vs.\ \(6.95\) units in deep recharge). This is the ``same energy, less time'' story, and it is visible in panel (b) of Figures~\ref{fig:partial-cross} and \ref{fig:deep-cross}. Re-optimizing the discrete plan is a distinct effect: in the partial-recharge scenario, the nonlinear optimum buys \emph{more} energy (\(10.17\) vs.\ \(3.35\) kWh), grows the mid-plan recharge from \(2.29\) to \(4.52\) units, and admits \(r_4\) into Trip 2; the \$76.99 profit gap is paid for by the extra charging, not by doing the same plan faster. In the deep-recharge scenario, re-optimization does nothing: the linear plan was already on the feasible frontier.

\section{Limitations}

The bilinear form \(E_{ksr}=p_{ksr}\theta_{ksr}\) in (35) is replaced by a piecewise-linear map from charging time to charged energy; this is what keeps the integrated MIP tractable but also means the implementation no longer carries the original relation exactly. The test instances are synthetic, generated on a small Euclidean grid, and the experiments are intentionally small because of the size-limited Gurobi license: the deep-recharge story would likely be sharper at \(R=6\), but that size currently exceeds the license budget once the initial-station decision layer is active. The model is also static and does not address dynamic request arrivals or stochastic travel times.

\section{Conclusion}

We formulate the static E-AFT routing and inter-trip charging problem as a single MIP, replace the bilinear charging relation by a station-specific piecewise-linear map wrapped in an indicator constraint, and compare linear and nonlinear charging on three synthetic scenarios. The two assumptions diverge only in the partial-recharge case, where the nonlinear curve enables one additional served request and a \$76.99 profit gap by buying more energy rather than by doing the same plan faster; they agree in the baseline and deep-recharge cases and differ there only in the charging time needed to acquire the same energy. The opposite regime, in which the linear model would commit to a deep recharge that nonlinear charging refuses, requires an \(R=6\) instance and is left for the next phase once a larger solver license is available.
