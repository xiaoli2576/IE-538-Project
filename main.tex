\begin{abstract}
\noindent
We formulate a static multi-trip routing and inter-trip charging problem for an Electrified-Autonomous Flexible Transit (E-AFT) fleet and solve it as a single mixed-integer program with explicit multi-station choice. The bilinear charging relation \(E_{ksr}=p_{ksr}\theta_{ksr}\) is replaced by a station-specific piecewise-linear map from charging time to charged energy, so the integrated model stays linear without sacrificing the tapering shape that drives the operational difference. On a 7-request single-bus deep-recharge instance the linear charging assumption commits to 35.81 kWh of inter-trip recharging in 24.47 minutes; replaying the same charge-time bookings against the nonlinear curve delivers only 31.55 kWh -- a 4.26 kWh shortfall that propagates through battery propagation and renders the discrete plan \emph{infeasible}: linear reaches an objective of 165.51 with five served requests, but the best honest nonlinear plan serves only four at 109.07. The 56.44 gap is therefore not a tuning artefact but a regime where the linear approximation produces an unexecutable plan. The same pathology shows up under the multi-bus configuration (\(|K|=2,\,R=2,\,|Q|=8\)) at a smaller magnitude (gap 6.24, again with an infeasible linear plan). We position this as a state-dependent boundary: the simplified charging assumption is safe when capacity is comfortable relative to demand, and misleading when it is tight enough to push an inter-trip recharge into the slow tail of the taper.
\end{abstract}

\section{Problem Description}

Electrified-Autonomous Flexible Transit (E-AFT) is an emerging and promising transportation mode for passenger travel. It allows passengers to submit travel requests online before traveling. They can detail their travel origins, destinations, and acceptable pickup time. Based on passenger requests, E-AFT routes are generated by selecting the appropriate stops to pick up and drop off passengers. Compared to traditional electric buses, E-AFT has a limited battery capacity that constrains traveling range. The smaller battery capacities make them more sensitive to routing planning to serve passenger requests, which requires further research on the optimization of transportation routing planning. In addition, the majority of current research assumes charging only happens at fixed depots \cite{gao2025integrated}. Based on these research gaps, Jiang et al. (2026) \cite{jiang2026two} developed a two-stage optimization approach for dynamic routing and charging scheduling. They assume that charging is restricted to happen between trips where no passengers are on board. They also assume a linear charging speed for vehicle charging. However, the charging speed can be nonlinear in reality \cite{yi2025charging, montoya2017electric}. Building on \cite{jiang2026two}, this project explores a static version of the trip routing and charging scheduling problem with a nonlinear charging profile and an explicit choice among multiple charging stations. The setting is illustrated in Figure~\ref{fig:concept}. Each bus moves from one stop to another to pick up and drop off passengers, which forms a trip. After all passengers drop off, the bus moves to a charging station to recharge. It then moves to the first bus stop of the next trip. The objective is to decide which passenger requests will be served by each bus, along with the route each bus follows within energy-consumption constraints. The nonlinear aspect of the project arises from the charging process, where the charging rate varies with the battery state instead of remaining constant. As a result, routing, timing, and charging decisions are tightly coupled.

\section{Mathematical Formulation}

Mathematical notation is summarized in Table~\ref{tab:notation}. Compared with the current study \cite{jiang2026two}, this project focuses on a static version in which routing and charging decisions are optimized once for the entire planning horizon. Instead of solving the problem in two heuristic stages, we formulate it in one optimization problem. In addition, this project examines the situation where the charging speed is nonlinear. For simplicity, several components of the original setting are omitted.

The objective function is the operational profit. It includes the revenue from served passenger requests, the fixed vehicle deployment cost, the penalty cost for unserved requests, the routing cost, and the charging cost.

Constraints (2) and (3) describe whether each request is served by a vehicle or not served. Constraints (4) and (5) ensure that the trip is formed, as at least one request is served during the trip. A subsequent trip can only be formed after the current trip is formed. Constraints (6)--(9) ensure that the pick-up and drop-off nodes are visited exactly once if the request is assigned to the trip. Constraint (10) ensures the flow conservation inside each trip. Constraints (11)--(14) define the first and the last node visited in trips by node connections. When there is no incoming arc for a node that is in the trip, it is the first node visited in the trip. If there is no outgoing arc for a node that is in the trip, it is the last node visited in the trip. Constraints (15)--(18) describe time constraints. Constraints (15) and (16) ensure that the arrival time at the pick-up node is within the acceptable time range. Constraint (17) ensures the drop-off time is later than the pick-up time. Constraint (18) describes within-trip time propagation. Constraints (19)--(21) represent the passenger-load changes as the bus picks up and drops off passengers along the way. Constraints (22)--(25) enforce the trip start and end time to equal the arrival time of the first and the last node of the trip. Constraint (26) ensures the arrival time of the starting node of the next trip is later than the arrival time of the last node of the current trip, plus the time traveling between nodes and a charging station, and the charging duration. Constraints (27) and (28) represent the change in battery level. Constraints (29)--(33) represent trip-level battery boundary constraints. Constraint (34) ensures that charging happens only between trips. Constraint (35) presents the relationship between charging power and charging time. Constraints (36) and (37) define the battery-level transitions between trips. Constraints (38)--(40) apply battery boundaries to the battery level along the trip. Constraints (41)--(51) pertain to the binary or non-negative variables. Among these constraints, (35) is a bilinear form that is naturally nonlinear, and the charging power \(p_{ksr}\) can itself be nonlinear.

\section{Implementation and Simplifications}

The model is implemented in Python with Gurobi.

\subsection{Why a Piecewise-Linear Charging Map Instead of the Bilinear Form}

Constraint (35), \(E_{ksr}=p_{ksr}\theta_{ksr}\), is bilinear in two continuous decision variables. Keeping this relation in its bilinear form inside the integrated model forces the solver into a mixed-integer quadratically constrained program (MIQCP) whose spatial branching interacts with the already large binary route-and-assignment space; in a size-limited license regime such as ours, this combination does not scale past two to three requests.

A more conservative relaxation -- McCormick envelopes -- would keep the problem MILP but yield only a convex outer approximation whose gap on tapering charging curves is driven by the largest product term, so solution quality is sensitive to the tightness of the battery-capacity bounds and the charging-time bounds used to build the envelope.

The formulation we adopt takes a different route. Physically, the quantity \(p_{ksr}\) is not an independent control; for a given battery and charger, the charging speed depends on the current state of charge by the battery management system. The published fast-charge profiles for lithium-ion EV batteries follow a CC-CV pattern whose energy-versus-time curve is well approximated by three to four affine segments \cite{yi2025charging, montoya2017electric}. Our model therefore keeps the \emph{physically observable} quantity -- the total energy \(E_{ksr}\) acquired as a function of time \(\theta_{ksr}\) -- and drops \(p_{ksr}\) entirely as a redundant intermediate variable. The resulting relation is encoded as \(\mathrm{PWL}\bigl(\theta_{ksr}; \theta^{(1)}_s,\ldots,\theta^{(P)}_s; E^{(1)}_s,\ldots,E^{(P)}_s\bigr)\) via Gurobi's \texttt{addGenConstrPWL}. On a segmented CC-CV-style curve this representation is exact at the breakpoints and interpolates linearly between them. Linear charging corresponds to evenly spaced breakpoints with constant slope; nonlinear (tapering) charging uses concave, decreasing-slope breakpoints. Station differences are expressed by scaling the time axis of the breakpoints, so a ``hub'' station delivers the same energy in less time than a ``depot.'' Figure~\ref{fig:charging-profiles} shows both base curves and the three station-specific variants.

\subsection{Tractability Adjustments}

Three further changes keep the integrated formulation tractable under a size-limited Gurobi license. The piecewise-linear charging relation is wrapped in an indicator on \(z_{ksr}\) so that the breakpoint model is inert when a station is not chosen; the redundant visit indicator \(v_{kir}\) is substituted by the corresponding assignment variable; and the initial dispatch station is promoted to a decision variable, so the first trip of each bus selects its origin together with the rest of the plan. Solver-side adjustments (per-constraint tightened big-\(M\), hierarchical multi-objective with profit primary and completion time secondary, and lex-order symmetry breaking across identical buses) are documented in the code repository.

\section{Computational Study}

The headline configuration uses one bus, three trip slots, two charging stations, and seven passenger requests, generated from Euclidean distances on a small synthetic grid. We added a multi-bus instance (two buses, two trip slots, eight requests) to check whether the conclusions carry to a fleet setting; that experiment is reported in Section~\ref{sec:multibus}.

The three scenarios share the same 7-request instance and differ only in the pickup time-window tightness applied to a subset of requests:
\begin{itemize}
    \item \textbf{Baseline} -- all seven windows are loose ($\approx 55$ minutes wide). Both charging assumptions are expected to serve all seven requests; this is the sanity check.
    \item \textbf{Partial recharge} -- requests \(r_3\) and \(r_4\) are pulled earlier and tightened to roughly 10-minute windows. The bus must either fit the squeezed pair into one trip or drop one of them. Here the recharge between trips is moderate, so the comparison probes the early (fast) portion of the taper.
    \item \textbf{Deep recharge} -- \(r_3\), \(r_4\), and \(r_5\) are tightened back-to-back. Serving all of them across the second and third trips requires a substantial recharge between trips and stresses the slow tail of the taper.
\end{itemize}

The deep-recharge scenario is the one that drives the main result. Once the new academic license replaced the size-limited pip license, the practical solve ceiling rose past two thousand variables and we increased \(|Q|\) from five to seven and \(|R|\) from two to three. With the new headroom the deep-recharge regime stops being benign and starts producing the asymmetry we expected from the original proposal.

In the \textbf{baseline} case, both assumptions serve all seven requests at the same objective of 353.64. The energy acquired is identical (35.95 kWh), but the nonlinear curve takes 41.27 minutes against the linear model's 24.57 -- because at this volume the recharge stretches into the slow tail of the taper, where extra time buys very little extra energy. The lesson is the simple one: tapering charging is not uniformly faster than constant-rate charging. It is faster only on the early segment.

In the \textbf{partial-recharge} case, both assumptions collapse to the same plan (objective 194.23, five served) and differ only in charge time (17.15 vs.\ 14.04 minutes for the same 25.10 kWh). At \(|Q|=7\), the squeezed pair \(\{r_3,r_4\}\) and the loose request \(r_5\) cannot all fit into the available trip budget under either model, and the choice of which to drop is dictated by routing geometry rather than charging speed. Partial-recharge therefore stops being the headline at this scale: the moderate-recharge regime does not separate the two assumptions when the time windows are interior to a single trip slot.

In the \textbf{deep-recharge} case, the two assumptions finally come apart. Linear reaches an objective of 165.51 by serving \(\{r_1,r_2,r_3,r_4,r_7\}\): it commits to a 35.81 kWh recharge over 24.47 minutes, betting that constant-rate charging supplies enough energy in the available inter-trip gap to make the third trip feasible. Nonlinear refuses that plan and instead falls back to four served requests \(\{r_1,r_2,r_4,r_5\}\) at 109.07, with a much smaller 23.00 kWh / 11.62-minute recharge. The 56.44 gap is therefore not a small numerical wobble -- it is a substantive disagreement on the served set. Section~\ref{sec:cross-eval} shows that the linear plan is in fact unexecutable under the nonlinear curve.

\subsection{Problem Size and Runtime}

Model size is dominated by the arc variables \(x_{kijr}\) and the first/last node indicators, which scale as \(|K|\,|N|^{2}|R|\) with \(|N|=2|Q|\) service nodes. Moving from \(|Q|=4,\,|R|=2\) to \(|Q|=7,\,|R|=3\) increases this dominant term by roughly 4.6\(\times\) per bus. With Gurobi 11 default settings the single-bus configuration solves to optimality in 1.8 seconds for the baseline, 3.3--3.9 seconds for partial recharge, and 2.3--2.7 seconds for deep recharge -- all well below our 300-second time limit. The linear and nonlinear modes share the same model size because each station carries the same number of breakpoints; the runtime differences between modes are within noise. The multi-bus configuration in Section~\ref{sec:multibus} grows by roughly 1.6\(\times\) in arc variables and takes 14--59 seconds depending on the scenario.

\subsection{Structural Changes in the Optimal Plans}

The deep-recharge case is where the discrete plans actually diverge. Linear serves \(\{r_1,r_2,r_3,r_4,r_7\}\) and routes the late \(r_7\) into a third trip, which only fits if the inter-trip recharges supply enough energy to clear the long \(r_7\) leg. At a constant rate over the available inter-trip windows, those recharges total 35.81 kWh in 24.47 minutes -- feasible on paper. Nonlinear, with the slow tail of the taper, cannot collect the same energy in the same windows. It serves only four requests \(\{r_1,r_2,r_4,r_5\}\) with a much smaller 23.00 kWh / 11.62-minute total recharge, dropping \(r_3,r_6,r_7\).

In the partial-recharge case, both models pick the same plan and the same drop set, so the structural change reduces to charging time only. In the baseline case the structure is also identical; the divergence is again in charging time, this time with the nonlinear curve charging \emph{longer}.

\subsection{Station Selection}

Across all runs the solver dispatches from \texttt{depot} and recharges between trips at \texttt{hub}. The hub combines favorable repositioning geometry with the fastest time-scaling factor (0.82 vs.\ 1.08 at the depot), so both the mid-plan station and the initial dispatch station end up actively selected rather than left at default. The fact that linear and nonlinear pick the \emph{same} stations but disagree on what is feasible there is part of why the deep-recharge gap is interpretable: it isolates the charging-curve effect from the station-choice effect.

\subsection{Cross-Evaluation: Is the Linear-Optimal Plan Still Good Under Nonlinear Charging?}
\label{sec:cross-eval}

We then test whether a plan optimized under the simplified linear assumption remains valid once the charging process is modeled more realistically. The cross-evaluation has three stages: (i) solve under linear charging; (ii) freeze the discrete plan (trip activations, routing arcs, station choices, first/last indicators) and re-solve the continuous timing and charging variables under nonlinear charging; (iii) re-optimize everything under nonlinear charging.

In the \textbf{partial-recharge} scenario the three stages collapse onto each other: the linear-optimal plan stays feasible under the nonlinear curve at the same objective of 194.23 with 25.10 kWh acquired in 14.04 minutes (vs.\ 17.15 minutes under the linear curve), and a free re-optimization under nonlinear returns the same plan. The nonlinear curve here just charges faster on the early segment.

The \textbf{deep-recharge} scenario is the substantive cross-evaluation. The linear-optimal plan books 24.47 minutes of inter-trip charging at the hub station, distributed across two recharges (11.35 min in the first and 13.12 min in the second), and counts on 35.81 kWh of acquired energy. Replaying those exact charge-time bookings against the nonlinear curve at the same station -- using the piecewise-linear breakpoints actually fitted into the model -- delivers only 31.55 kWh, a 4.26 kWh shortfall (\(15.47\) vs.\ \(16.61\) kWh on the first recharge and \(16.09\) vs.\ \(19.20\) kWh on the second; full per-trip numbers in \texttt{results/deep\_recharge\_infeasibility\_diagnostic.json}). With the discrete plan fixed (trip activations, routing arcs, station choices, first/last indicators), the missing 4.26 kWh propagates through the battery-propagation constraints and the target-mode solve returns \emph{infeasible}. A free nonlinear re-optimization is the next-best honest answer; it serves four requests at 109.07 with a much smaller recharge total of 23.00 kWh / 11.62 minutes. The 56.44 gap between the linear-optimal and the nonlinear-optimal objectives therefore measures the cost of trusting a charging assumption that the physical battery cannot match.

\subsection{Multi-Bus Robustness Check}
\label{sec:multibus}

A natural concern about the headline result is that single-bus capacity is unrealistically tight. We therefore re-ran the deep-recharge scenario with two homogeneous buses, two trip slots per bus, and eight requests (instances were generated with the same seed and Euclidean grid, but with the additional bus and request added).

The qualitative result carries over but at a smaller magnitude. In direct comparison the linear assumption reaches 300.22 by serving seven of the eight requests, with one bus committing to an 18.18 kWh inter-trip recharge to fit a three-request second trip. Under the nonlinear curve the best plan (also seven served) returns 293.98, a gap of 6.24. The cross-evaluation again flags the linear plan as \emph{infeasible} under the nonlinear curve: the deep recharge that fits at constant rate cannot be acquired inside the same inter-trip window once the tail of the taper is enforced. Re-optimization under nonlinear instead spreads load across the two buses with smaller per-trip recharges and keeps seven served at 293.98.

The pattern we observe in both the single-bus (\(|K|=1,\,|R|=3\)) and the two-bus (\(|K|=2,\,|R|=2\)) configurations is consistent: the asymmetry shows up when capacity is tight enough that fitting all promised requests forces a substantial inter-trip recharge into the slow tail of the taper. We expect that further loosening capacity (more buses, more trip slots, or both) would dilute this regime, since each trip becomes shorter and each inter-trip recharge stays inside the early segment where the two charging assumptions barely disagree -- but we have not run a saved sweep over those configurations and do not claim a quantitative boundary here.

\section{Limitations}

The bilinear form \(E_{ksr}=p_{ksr}\theta_{ksr}\) in constraint (35) is replaced by a piecewise-linear map from charging time to charged energy. This keeps the integrated formulation a MIP rather than a MIQCP and is what makes the deep-recharge experiment tractable on a laptop, but it does mean the implementation no longer carries the original bilinear relation exactly. The test instances are synthetic, generated on a small Euclidean grid, and we do not benchmark against published transit datasets. The model is also static; it does not handle dynamic request arrivals or stochastic travel times. Finally, the symmetry-breaking we apply across identical buses is a lex-order constraint on bus usage, which is sound but loose; a tighter symmetry break would likely accelerate the multi-bus experiments further.

\section{Conclusion}

We formulate the static E-AFT routing and inter-trip charging problem as a single MIP and replace the bilinear charging relation by a station-specific piecewise-linear map guarded by indicator constraints. On a 7-request single-bus instance, the linear and nonlinear charging assumptions agree in the baseline and partial-recharge regimes, and differ only in charging time. The deep-recharge regime is where they come apart: linear commits to 35.81 kWh of inter-trip recharging that the nonlinear curve, in the same booked time, only supplies as 31.55 kWh -- a 4.26 kWh shortfall that breaks the discrete plan's feasibility check. The honest nonlinear plan instead serves four requests at 109.07. This 56.44 gap is the central result -- it is not a tuning artefact, it measures the cost of relying on a charging approximation that the physical battery cannot match. The same pattern shows up at smaller magnitude in the two-bus, eight-request configuration. We read the takeaway as state-dependent rather than monotone: the linear approximation is safe when capacity is comfortable relative to demand, and misleading when capacity is tight enough to force a deep recharge into the slow tail of the taper.
